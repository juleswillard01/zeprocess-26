================================================================================
SECURITY FEATURES
================================================================================

✓ Non-root containers (appuser:1000)
✓ SSL/TLS with Let's Encrypt (auto-renewal)
✓ UFW firewall (deny by default, allow 22/80/443)
✓ SSH key authentication only
✓ Secrets in .secrets/ (gitignored, never committed)
✓ Database isolated to internal network
✓ Redis isolated to internal network
✓ Rate limiting on all endpoints
✓ Security headers (HSTS, X-Frame-Options, CSP, etc.)
✓ Health check monitoring
✓ Auto-restart on failure with exponential backoff
✓ Log rotation and archival

DEPLOYMENT STEPS
================================================================================

PHASE 1: VPS Setup (30 min)
  1. Provision Hetzner VPS (Ubuntu 24.04, 4core/8GB)
  2. Update system packages
  3. Install Docker + Docker Compose
  4. Create quixai user
  5. Clone repository

PHASE 2: Secrets & Config (15 min)
  1. Run ./infra/scripts/init-secrets.sh
  2. Configure .env file
  3. Add .secrets/ to .gitignore

PHASE 3: Firewall (10 min)
  1. Run ./infra/scripts/setup-firewall.sh
  2. Verify ports 22, 80, 443 open

PHASE 4: Docker Compose (20 min)
  1. cd infra/docker && docker-compose up -d
  2. Wait for all containers healthy
  3. Test health endpoints

PHASE 5: SSL/TLS (15 min)
  1. Install certbot
  2. Obtain certificate with certbot
  3. Copy to /opt/mega-quixai/infra/ssl/
  4. Restart nginx

PHASE 6: Systemd Agents (15 min)
  1. Copy systemd files to /etc/systemd/system/
  2. systemctl daemon-reload
  3. systemctl start mega-quixai.target
  4. Verify agents running

PHASE 7: Backups (10 min)
  1. Create /var/backups/mega-quixai/
  2. Setup cron job for backups
  3. Test backup/restore

PHASE 8: Verification (10 min)
  1. Run ./infra/scripts/post-deploy-check.sh
  2. Verify all health checks pass

PHASE 9: Monitoring (20 min, optional)
  1. Setup Grafana dashboard
  2. Configure alerting rules

PHASE 10: CI/CD (10 min)
  1. Configure GitHub secrets
  2. Test deployment pipeline

TOTAL TIME: 2-3 hours

================================================================================
BACKUP & RECOVERY
================================================================================

BACKUP STRATEGY:
  Frequency: Daily at 2 AM UTC
  Retention: 30 days
  Location: /var/backups/mega-quixai/
  Format: gzipped SQL dump
  Size: ~50-100 MB per backup

RECOVERY PROCEDURE:
  1. Stop API: systemctl stop mega-quixai.service
  2. Restore: ./scripts/restore.sh <backup-file.sql.gz>
  3. Start API: systemctl start mega-quixai.service

TIME TO RECOVER: ~5 minutes

================================================================================
TROUBLESHOOTING
================================================================================

AGENT STUCK IN RESTART LOOP:
  $ journalctl -u mega-quixai-agent-seduction -n 50
  $ curl http://localhost/health
  $ docker-compose logs postgres

POSTGRESQL CONNECTION REFUSED:
  $ docker exec mega-quixai-postgres pg_isready -U quixai_user
  $ docker-compose restart postgres

DISK SPACE FULL:
  $ find /var/log -name "*.log" -mtime +30 -delete
  $ docker system prune -a --volumes

MEMORY LEAK:
  $ docker stats
  $ docker-compose restart api

PORT ALREADY IN USE:
  $ lsof -i :3000
  $ kill -9 <PID>
