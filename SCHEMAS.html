Sequence API URSSAF — Appels Techniques

Diagramme de sequence : OAuth2 + REST + Swan GraphQL.
100% Scroll = zoom | Drag = pan | Double-clic = plein ecran 
sequenceDiagram
    actor Jules
    participant App as SAP-Facture
    participant Sheets as Google Sheets
    participant URSSAF as API URSSAF
    participant Client as Client (eleve)
    participant Swan as Swan/Indy

    Note over App,URSSAF: Phase 1 — Authentification
    App->>URSSAF: POST /oauth/token {client_id, client_secret}
    URSSAF-->>App: {access_token, expires_in}

    Note over Jules,App: Phase 2 — Inscription client
    Jules->>App: Nouveau client (nom, email, adresse)
    App->>URSSAF: POST /particuliers {identite, email, adresse}
    URSSAF-->>App: {id_technique}
    App->>Sheets: Ecrire dans onglet Clients

    Note over Jules,URSSAF: Phase 3 — Soumission facture
    Jules->>App: Creer facture (client, heures, tarif, dates)
    App->>App: Valider payload + Generer PDF
    App->>Sheets: Ecrire dans onglet Factures (BROUILLON)
    App->>URSSAF: POST /demandes-paiement {id_client, montant, nature, dates}
    URSSAF-->>App: {id_demande, statut: CREE}
    App->>Sheets: Maj statut = SOUMIS

    Note over URSSAF,Client: Phase 4 — Validation client
    URSSAF->>Client: Email : validez votre facture
    Client->>URSSAF: Validation en ligne

    Note over App,URSSAF: Phase 5 — Polling statut (cron 4h)
    loop Toutes les 4 heures
        App->>URSSAF: GET /demandes-paiement/{id}
        URSSAF-->>App: {statut: VALIDE/PAYE/REJETE}
        App->>Sheets: Maj statut dans onglet Factures
    end

    Note over App,Jules: Phase 5b — Reminder T+36h
    App->>App: Check 36h sans validation
    App->>Jules: Email : relancer client X

    Note over URSSAF,Swan: Phase 6 — Paiement
    URSSAF->>Swan: Virement 100% du montant facture
    Note right of Client: Client paye son 50%\ndirectement a l'URSSAF\n(jamais a Jules)

    Note over App,Sheets: Phase 7 — Rapprochement
    App->>Swan: GET /transactions
    Swan-->>App: Liste transactions
    App->>Sheets: Import dans onglet Transactions
    Note over Sheets: Onglet Lettrage : formules matching auto
    Note over Sheets: Onglet Balances : soldes calcules

Rapprochement Bancaire & Lettrage — Swan / URSSAF

Lettrage dans Google Sheets : formules matching factures (onglet Factures) avec virements (onglet Transactions).
100% Scroll = zoom | Drag = pan | Double-clic = plein ecran 
flowchart TD
    START["Pour chaque facture PAYEE\ndans onglet Factures"] --> WINDOW["Filtrer transactions URSSAF\ndate_paiement +/- 5 jours"]
    WINDOW --> SEARCH["Chercher 1 transaction :\nMontant = 100% facture\nLibelle contient URSSAF"]
    SEARCH --> FOUND{"Transaction\ntrouvee ?"}
    FOUND -- Oui --> SCORE["Score confiance :\nmontant exact = +50\ndate < 3j = +30\nlibelle URSSAF = +20"]
    SCORE --> HIGH{"Score >= 80 ?"}
    HIGH -- Oui --> AUTO["LETTRE AUTO\nEcrire facture_id dans transaction\nStatut = LETTRE"]
    HIGH -- Non --> SUGGEST["A VERIFIER\nSurligner en orange\nJules confirme manuellement"]
    FOUND -- "Non" --> NOMATCH["PAS DE MATCH\nAttendre le virement URSSAF\nSurligner en rouge"]
    AUTO --> BALANCE["Mettre a jour\nonglet Balances"]
    SUGGEST --> BALANCE

Machine a Etats — Cycle de Vie Facture

Tous les statuts possibles et transitions entre eux.
100% Scroll = zoom | Drag = pan | Double-clic = plein ecran 
stateDiagram-v2
    [*] --> BROUILLON : Jules cree la facture
    BROUILLON --> SOUMIS : Envoi API URSSAF
    BROUILLON --> ANNULE : Jules annule
    SOUMIS --> CREE : URSSAF accepte le payload
    SOUMIS --> ERREUR : Payload invalide ou API down
    ERREUR --> BROUILLON : Corriger les champs
    CREE --> EN_ATTENTE : Email envoye au client
    EN_ATTENTE --> VALIDE : Client valide sur portail URSSAF
    EN_ATTENTE --> EXPIRE : Delai 48h depasse
    EN_ATTENTE --> REJETE : Client refuse
    EXPIRE --> BROUILLON : Re-soumettre
    REJETE --> BROUILLON : Corriger si besoin
    VALIDE --> PAYE : URSSAF effectue le virement
    PAYE --> RAPPROCHE : Match avec transactions Swan
    RAPPROCHE --> [*] : Cycle termine
    ANNULE --> [*] : Cycle termine
    note right of BROUILLON
        PDF genere
        Pas encore envoye a URSSAF
        Modifiable par Jules
    end note
    note right of CREE
        URSSAF a recu
        Client doit valider sous 48h
    end note
    note right of EN_ATTENTE
        Timer 48h actif
        Reminder a T+36h
    end note
    note right of PAYE
        Virement en cours
        Verifier compte Swan
    end note
    note right of RAPPROCHE
        Tout est OK
        Comptabilite clean
    end note
