# Refactor Guide

Analyser le code et identifier les opportunités de refactoring.

1. Déléguer au refactor-guide agent
2. Scanner pour : dead code, duplication, fonctions > 30 lignes, nesting > 3
3. Vérifier que tous les tests passent AVANT de toucher quoi que ce soit
4. Proposer un plan de refactoring priorisé
5. Chaque refactoring = un commit séparé
6. Vérifier les tests APRÈS chaque changement

Focus sur : $ARGUMENTS
