# Code Review

Revue de code complète sur les changements en cours.

1. `git diff` pour voir tous les changements
2. Déléguer au code-reviewer pour qualité
3. Déléguer au security-auditor pour sécurité
4. Déléguer au performance-reviewer si changements I/O ou API
5. Agréger tous les rapports
6. Produire un verdict global avec actions requises
